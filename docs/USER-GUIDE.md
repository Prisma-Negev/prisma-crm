# Prisma Connect — מדריך שימוש

מערכת ניהול חוקרים ואירועים עבור מרכז החדשנות פריזמה נגב, אוניברסיטת בן גוריון.

---

## התחברות

- גש לכתובת המערכת (למשל `https://prisma-connect.netlify.app`)
- **משתמש חדש:** לחץ "הרשמה" → מלא שם, מייל וסיסמה → קבל מייל אישור → לחץ על הקישור
- **משתמש קיים:** מייל + סיסמה → "כניסה"
- **SSO:** כניסה עם Google/Microsoft (אם הוגדר)

---

## דשבורד

המסך הראשי מציג:
- **סיכום מספרי** — כמה חוקרים, אירועים פתוחים, משימות ממתינות
- **גרף Pipeline** — כמה חוקרים בכל שלב (טרם פנייה, נשלח מייל, מעוניין וכו')
- **פעילות אחרונה** — הערות, משימות, ושינויים אחרונים

---

## חוקרים (Contacts)

### רשימת חוקרים
- טבלה עם סינון, מיון וחיפוש חופשי
- סינון לפי: מחלקה, עדיפות, תגיות, סטטוס
- מיון לפי: שם, תאריך הוספה, עדיפות

### הוספת חוקר
שדות עיקריים:
| שדה | הסבר |
|------|-------|
| שם פרטי / משפחה | שם מלא |
| תואר אקדמי | פרופ' מן המניין, פרופ' חבר, מרצה בכיר, ד"ר, פוסט-דוק, סטודנט |
| מחלקה | מחלקה ב-BGU (הנדסת תעשייה, מכונות, חשמל וכו') |
| מייל | כתובת מייל (אפשר כמה) |
| טלפון | מספר טלפון (אפשר כמה) |
| CRIS Profile | קישור לפרופיל ב-cris.bgu.ac.il — לחיצה פותחת בטאב חדש |
| תחום מחקר | תיאור חופשי של תחום המחקר |
| עדיפות | 1=דחוף (אדום), 2=גבוה (כתום), 3=בינוני (צהוב), 4=נמוך (אפור) |
| סקטור | תחום כללי (רובוטיקה, סייבר, AI וכו') |
| תגיות | תגיות חופשיות לסיווג |
| LinkedIn | קישור לפרופיל LinkedIn |
| רקע | מידע כללי על החוקר |

### כרטיס חוקר
בלחיצה על חוקר מוצג כרטיס מלא הכולל:
- כל הפרטים + קישור ישיר ל-CRIS
- צבע עדיפות ברור
- **היסטוריית הערות** — כל הפניות והתגובות
- **משימות** — מה עוד צריך לעשות
- **אירועים** — לאילו אירועים הוזמן

---

## מחלקות / מרכזי מחקר (Companies)

- מייצגות מחלקות אקדמיות ומרכזי מחקר ב-BGU
- מחלקות מוגדרות מראש: הנדסת תעשייה וניהול, הנדסת מכונות, הנדסת חשמל, מדעי המחשב, הנדסה ביו-רפואית, כימיה, הנדסת חומרים, הנדסה אזרחית, חינוך, פסיכולוגיה
- אפשר להוסיף מחלקות/מרכזים חדשים
- כל חוקר משויך למחלקה אחת

---

## אירועים (Deals) — לוח Kanban

### שלבי Pipeline
האירועים מנוהלים בלוח Kanban עם גרירה (Drag & Drop):

```
טרם פנייה → נשלח מייל → מעוניין → אישר השתתפות → השתתף
                                                  → סירב
                                                  → לא ענה
```

### יצירת אירוע
| שדה | הסבר |
|------|-------|
| שם האירוע | למשל "כנס התאחדות התעשיינים 2026" |
| סוג | כנס תעשייה, פורום חינוך, סדנה, מפגש, וובינר, אחר |
| תיאור | פירוט על האירוע |
| תאריך צפוי | תאריך האירוע |
| חוקרים | שיוך חוקרים לאירוע |

### שימוש
- גרור כרטיס חוקר בין עמודות כדי לעדכן סטטוס
- לחץ על כרטיס לפרטים מלאים
- הלוח מציג סיכום כספי (אם רלוונטי) וכמות בכל שלב

## משימות (Tasks)

### סוגי משימות
- מייל ראשוני (Cold Email)
- מעקב (Follow-up)
- שיחת טלפון
- פגישה
- בדיקת CRIS
- פנייה למזכירות
- אישור השתתפות
- מייל תודה

### יצירת משימה
1. מתוך כרטיס חוקר → "משימה חדשה"
2. בחר סוג, תאריך יעד, ותיאור
3. המשימה תופיע גם בדשבורד וגם בכרטיס החוקר

### סימון כהושלמה
- לחץ על ה-checkbox ליד המשימה
- המשימה עוברת לרשימת "הושלמו"

---

## הערות (Notes)

- הוספה מכרטיס חוקר → "הערה חדשה"
- **סטטוסים:** פנייה ראשונית (כחול), בתהליך (צהוב), תגובה חיובית (ירוק), תגובה שלילית (אדום)
- אפשר לצרף קבצים (מסמכים, תמונות)
- ההערות מוצגות בסדר כרונולוגי — יוצרות "יומן אינטראקציות"

---

## תגיות (Tags)

- יצירת תגיות: הגדרות → תגיות → "תגית חדשה"
- לכל תגית: שם + צבע
- דוגמאות מומלצות: `רובוטיקה`, `AI`, `תעשייה 4.0`, `אנרגיה`, `חינוך`, `EdTech`, `כנס-תעשייה-2026`
- סינון חוקרים לפי תגיות ברשימה

---

## ייבוא נתונים (Import)

1. הכן קובץ CSV עם עמודות מתאימות (first_name, last_name, email וכו')
2. לך ל-**Import** בתפריט
3. העלה את הקובץ
4. מפה עמודות מה-CSV לשדות במערכת
5. בדוק תצוגה מקדימה → אשר ייבוא

**טיפ:** הקובץ `Prisma_CRM_FINAL_44_Researchers.xlsx` שכבר יש לך — המר ל-CSV וייבא.

## הגדרות (Settings)

- **כללי:** שם המערכת, לוגו, מטבע (ILS)
- **Pipeline:** הוספה/עריכת שלבי אירועים
- **סוגי אירועים:** הוספת קטגוריות
- **סוגי משימות:** התאמת סוגי משימות
- **צוות (Sales):** ניהול משתמשים — הוספה, הסרה, הרשאות
- **פרופיל:** עדכון שם, תמונה, סיסמה

## תהליך עבודה מומלץ — גיוס חוקרים לכנס

### שלב 1: הכנה
1. צור **אירוע חדש** (למשל "כנס התאחדות התעשיינים 10.2")
2. הגדר סוג: "כנס תעשייה"
3. הגדר תאריך צפוי

### שלב 2: זיהוי חוקרים
1. סנן חוקרים לפי **תגיות** רלוונטיות (רובוטיקה, תעשייה 4.0)
2. סנן לפי **עדיפות** 1-2 (גבוהה)
3. שייך חוקרים מתאימים לאירוע

### שלב 3: פניות
1. לכל חוקר — צור **משימה** מסוג "מייל ראשוני"
2. שלח מייל (תבנית Cold Email)
3. הוסף **הערה** עם סטטוס "פנייה ראשונית"
4. העבר באירוע ל-"נשלח מייל"

### שלב 4: מעקב
1. לאחר 3-5 ימים → משימה "מעקב" לחוקרים שלא ענו
2. עדכן הערות עם תגובות
3. העבר חוקרים ל-"מעוניין" / "סירב" / "לא ענה"

### שלב 5: אישורים
1. חוקרים שאישרו → משימה "אישור השתתפות"
2. העבר ל-"אישר השתתפות"
3. לאחר הכנס → העבר ל-"השתתף"
4. שלח מייל תודה

### שלב 6: סיכום
- בדוק בדשבורד: כמה חוקרים בכל שלב
- ייצא דוח לצוות/הנהלה

## פתרון בעיות

| בעיה | פתרון |
|-------|--------|
| לא מצליח להתחבר | בדוק שה-Supabase URL וה-anon key נכונים ב-Environment Variables |
| הנתונים לא נשמרים | בדוק ב-Supabase שה-RLS policies מוגדרות נכון |

## מידע טכני לפיתוח עתידי

- **Stack:** React 19 + TypeScript + Vite 7 + Supabase + react-admin 5 + shadcn/ui + Tailwind CSS 4
- **DB:** PostgreSQL (via Supabase) עם Row Level Security
- **Auth:** Supabase Auth (email, SSO)
- **API:** PostgREST (auto-generated REST API)
- **i18n:** ra-i18n-polyglot — עברית כברירת מחדל, תמיכה באנגלית וצרפתית
- **Tests:** Vitest + Playwright
- **Repo:** https://github.com/Prisma-Negev/prisma-crm
- **Legacy backup:** branch `legacy/v1` ב-Events-Crm repo
